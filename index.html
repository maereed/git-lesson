What is Git?

Command line tool

Version control

So you can manage a project, or set of files as they change over time.

Git stores this information in a data structure called a repository (repo)

Commit objects

References to these objects called HEADs

Commit Object

A set of files

A string that uniquely identifies each one

Head

A head is simply a reference to a commit object

Every repository get a default called master

At any given time, one head is selected as the current head. This head is aliased to HEAD, always in capitals.

A repository can contain any number of heads.
The unmodified and modified files are pretty self explanatory: the difference is whether or not you have made changes. An unmodified file is one that you have not touched, and it will not appear if you run `git status`. **The modified file is a file that has been changed but not added**. It will appear under the heading `Changes not staged for commit:` when you run `git status`. The staging area is where the files are added and ready to commit. So this happens after the **git add myfile.html** command we would run.

Moving files

talk about mv vs git mv and the problems that can occur

for example

git mv hello.html folder/
git status

you can do it move the files by: 

mkdir folder/
mv hello.html folder/
git add folder/hello.html
git rm hello.html

Moving files with git, we notify git about two things

1. The hello.html file was deleted.

2. The lib/hello.html file was created.

Explain that without `git mv` git doesn't realize that's the same file so... if anyone else adds to the file, and you've move it, the changes have to be made manually.
Removing files
